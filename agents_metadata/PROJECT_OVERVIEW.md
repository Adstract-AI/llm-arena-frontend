# LLM Arena – Project Overview

## Description

LLM Arena is a simple web-based platform designed to compare and explore different large language models (LLMs) through human interaction.

The system has two main interaction modes. Users can start a blind battle conversation and receive two anonymous responses generated by two different models. They can continue the conversation across multiple turns, see both anonymous answer streams evolve side by side, and then vote on which model performed better across the full conversation transcript. This enables blind, pairwise comparison based on real user preferences over a richer interaction than a single prompt-response exchange.

The project also includes a direct chat mode where users can choose a specific Vezilka model and chat with it normally. This gives users more control when they want to explore a single model's behavior outside the blind arena flow.

The main goal of this project is to evaluate **Macedonian fine-tuned LLMs**.

---

## Core Concept

- A user starts a battle by submitting a prompt.
- The backend sends the same prompt to two different LLMs.
- The frontend displays both anonymous answer streams as **Answer A** and **Answer B**.
- The user can continue the battle with additional prompts, and both models answer each turn.
- After reviewing the full conversation, the user selects:
  - A is better
  - B is better
  - Tie / both are equal
- The full conversation transcript and final vote are stored for later analysis.
- Separately, a user can open the chat mode, select a Vezilka model, and have a normal single-model conversation with persistent chat history.

This setup ensures **blind evaluation** and reduces bias.

---

## Tech Stack

- **Frontend**: React (simple UI for prompt input, response display, and voting)
- **Backend**: Django (API, business logic, model orchestration, data storage)
- **LLM Providers**:
  - OpenAI
  - Anthropic
  - Macedonian fine-tuned models (available via OpenAI-compatible API)

---

## System Architecture

### Frontend
- Prompt input field
- Two response panels (A and B) that grow across turns
- Ability to continue the battle with more prompts
- Voting controls for the full conversation
- Direct chat view for choosing and testing a Vezilka model
- Loading and error states

### Backend
- Receives a prompt and initializes a "battle"
- Selects two models (randomly or predefined)
- Queries both models with the same prompt on every turn
- Stores the full turn-by-turn transcript and metadata
- Returns anonymized transcript snapshots to the frontend
- Accepts and stores votes for the completed conversation
- Exposes a direct chat API for selected Vezilka models and stores chat sessions and messages

---

## Key Requirements

- **Blind comparison**: Users must not know which model generated which answer stream until after the vote.
- **Same prompt for both models on every turn**: Ensures fairness across the whole conversation
- **Randomized response order**: Prevents positional bias
- **Conversation-level voting**: The vote should represent the full conversation, not just the opening turn
- **Persistent storage**: All turns, responses, and votes must be saved
- **Separate direct chat mode**: Users should also be able to interact with a chosen Vezilka model outside the arena flow

---

## Goal

The system provides a simple but effective way to:
- Collect human preference data
- Compare local (Macedonian) vs global LLMs
- Analyze performance trends across models

This project focuses on conversation-level evaluation, not generic standalone chat.
At the same time, it also includes direct Vezilka chat so users can explore individual models with more control.
